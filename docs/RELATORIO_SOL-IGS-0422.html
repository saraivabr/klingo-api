TAREFA 02 — OCR + Agendamento Automático

Arquitetura entregue

Pipeline público implementado usando Claude Vision (Anthropic SDK) — sem dependência de Google Vision / AWS Textract. A IA lê a foto do pedido e extrai dados estruturados em JSON; o sistema cria uma escalação de atendimento para a equipe confirmar e agendar via fluxo Klingo existente.

Fluxo

  1. Paciente acessa https://<host>/enviar-pedido
  2. Preenche nome + WhatsApp e anexa foto do pedido médico (JPG/PNG/WEBP até 10MB)
  3. Frontend converte para base64 e chama POST /api/exam-requests/upload
  4. API envia à Claude Vision com prompt estruturado — extrai paciente, CPF, médico, CRM, exames, data, indicação clínica, nível de confiança
  5. Sistema tenta casar com paciente existente pelo telefone
  6. Cria escalation com reason exam_request_auto_ocr e prioridade conforme a confiança
  7. Paciente recebe protocolo + lista dos exames identificados. Equipe recebe escalação no painel para confirmar o agendamento

Arquivos criados/modificados

  • apps/api/src/routes/exam-requests.ts — rota pública (rate limit 10/min)
  • apps/api/src/server.ts — registro + bodyLimit 20MB
  • apps/api/package.json — dep @anthropic-ai/sdk
  • apps/booking/src/ExamUpload.tsx — tela pública de upload
  • apps/booking/src/main.tsx — dispatch de rota por URL
  • infra/nginx/default.conf — servir /enviar-pedido a partir do bundle booking

Defaults adotados (ajustáveis)

  • Modelo: claude-sonnet-4-5 (override via CLAUDE_VISION_MODEL)
  • Tamanho máx.: 10MB por imagem
  • Formatos: JPG, PNG, GIF, WEBP (PDF requer trabalho adicional)
  • Agendamento: confirmação manual pela equipe (via escalações) — evita bookings errados por OCR ambíguo. Pode evoluir para agendamento 100% automático quando houver tabela mapping exame→procedimento Klingo
