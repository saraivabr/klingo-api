5. Sistema de Filas (BullMQ)

7 filas de processamento assíncrono

📥
message-intake
Recebe mensagens do webhook, normaliza telefone, dedup com Redis lock, verifica rate limit
Concorrência: 10 Trigger: Webhook
🧠
ai-pipeline
Carrega contexto, chama Claude/OpenAI, executa tools (booking, RAG, escalation), transição de estado
Concorrência: 5 Trigger: Intake
📤
message-send
Envia texto via UAZAPI, botões interativos, typing indicator, reações com emoji
Concorrência: 10 Trigger: AI Pipeline
🔔
follow-up
Sequência pós-booking: lembrete, confirmação, pesquisa de satisfação
Concorrência: 3 Trigger: Post-booking
📊
analytics
Calcula métricas diárias: taxa de sucesso, tempo médio de resposta, sentimento
Concorrência: 2 Trigger: Real-time
🧹
booking-cleanup
Expira booking links antigos (>30 min), limpa dados temporários
Concorrência: 1 Cron: A cada 60 min
⏰
appointment-reminder
Lembrete diário de consultas do dia seguinte enviado via WhatsApp
Concorrência: 2 Cron: 18:00 BRT

6. Motor de IA

Persona "Julia" - recepcionista virtual com inteligência emocional

Componentes do AI Engine

prompts/system.ts
System prompt de 300+ linhas. Persona "Julia", regras de comunicação WhatsApp, dados da clínica, PNL e rapport
classifiers/intent.ts
17 intents via regex: greeting, booking, price, availability, complaint, urgency, farewell...
classifiers/escalation.ts
Detector de urgência médica (dor, febre, sangramento) e reclamações graves
rag/search.ts
Busca semântica via pgvector (embeddings 1536-dim). Knowledge base com FAQs da clínica
state-machine/machine.ts
Transições de estado baseadas em intent + contexto. 11 estados, regras de escalação
audio/transcribe.ts
Transcrição de áudios do WhatsApp (Whisper). Paciente pode falar em vez de digitar

8. Fluxo de Agendamento

Do pedido do paciente até a consulta confirmada

1
Paciente pede para agendar
IA detecta intent "appointment_booking", pergunta especialidade via botões interativos
Intent classifier → send_interactive_message tool
2
IA consulta disponibilidade
Busca slots livres na Klingo API por médico e especialidade selecionados
check_availability → Klingo getSmartSlots()
3
Gera link de agendamento
Cria token único com validade de 30 min, salva no PostgreSQL (booking_links)
generate_booking_link → token JWT, expires_at
4
Paciente abre o link
React Booking Widget exibe slots disponíveis com médico, data e horário
https://irb.saraiva.ai/agendar?token=xxx → React SPA
5
Paciente seleciona horário
Cria appointment no PostgreSQL, atualiza booking_link com status "used"
POST /api/booking/confirm → appointments table
6
Confirmação via WhatsApp
IA envia mensagem de parabéns + botão "Adicionar à agenda" (Google Calendar)
message-send queue → UAZAPI buttons → calendar.google.com
7
Lembrete automático
Worker envia lembrete no dia anterior à consulta, às 18h (BRT)
appointment-reminder cron → UAZAPI text + buttons

10. Estrutura do Monorepo

│ │ └── src/ → routes/, services/, middleware/, websocket/
